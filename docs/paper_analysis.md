## 1. システムアーキテクチャ全体像

### 1.1 3段階パイプライン

```
ソース音声 (16kHz)
       │
       ▼
┌──────────────────────────┐
│      Destylizer          │  スタイル情報を除去し、言語内容のみを抽出
│  HuBERT-Large 第18層     │
│  → Conformer ×6          │
│  → コンテンツ特徴量 fc   │  ※推論時はFSQを通さず、直前の連続表現を使用
└──────────┬───────────────┘
           │ fc ∈ R^{768 x T}  (50Hz)
           │
ターゲット音声 (16kHz)
       │
       ├─→ メルスペクトログラム → コンテキスト
       └─→ スタイルエンコーダ (WavLM-TDNN) → スタイル埋め込み e
           │
           ▼
┌──────────────────────────┐
│      Stylizer (DiT)      │  コンテンツ＋ターゲットスタイルからメル生成
│  16層トランスフォーマー   │
│  CFM + adaLN-Zero        │
│  オイラーサンプリング     │
│  (NFE=16)                │
└──────────┬───────────────┘
           │ メルスペクトログラム ∈ R^{100 x T}  (50Hz)
           ▼
┌──────────────────────────┐
│  ボコーダ（因果的Vocos）  │  メル → 波形
└──────────┬───────────────┘
           │
           ▼
      変換音声 (16kHz)
```

### 1.2 統一フレームレート

全コンポーネントが **50Hz** で統一されている:
- HuBERT: 50Hz出力 (20ms/フレーム)
- Destylizerコンテンツ特徴量: 50Hz
- メルスペクトログラム: ホップサイズ320 @ 16kHz → 50Hz
- Stylizer DiT: 50Hz
- ボコーダ: 50Hz入力 → 16kHz出力

### 1.3 推論時のデータフロー

1. ソース音声 → Destylizer → コンテンツ特徴量 `fc_source`
2. ターゲット音声 → Destylizer → `fc_target`（最終的に破棄）
3. ターゲット音声 → メル計算 → `x1_target`（コンテキスト）
4. ターゲット音声 → スタイルエンコーダ → スタイル埋め込み `e`
5. `fc_target` と `fc_source` を時間軸で結合、ターゲット部分は非マスク・ソース部分はマスク
6. Stylizer（DiT）がマスク領域のメルを生成（CFM + オイラーサンプリング、NFE=16、CFG α=2）
7. ボコーダ（因果的Vocos）で波形に変換

## 2. Destylizer（コンテンツ-スタイル分離）

### 2.1 アーキテクチャ

```
音声入力 → HuBERT-Large-ASR（凍結、第18層）→ Conformerブロック（×6）
         → [コンテンツ特徴量 fc] → FSQ射影（下）→ FSQ量子化 → FSQ射影（上）→ ASRデコーダ → テキスト
```

| コンポーネント | 仕様 |
|---|---|
| 入力特徴量 | HuBERT-Large-ASR（`hubert-large-ls960-ft`）第18層出力 |
| HuBERT凍結 | 学習時は凍結（ストリーミング版では解凍＋チャンク因果化） |
| Conformerブロック | 6層、ALiBi位置エンコーディング |
| ASRデコーダ | 4層トランスフォーマーデコーダ（隠れ層768、FFN 3072） |
| 出力サンプリングレート | 50 Hz |

**重要**: 推論時はASRデコーダを使用せず、FSQ **直前** の連続表現（コンテンツ特徴量 `fc`）のみをStylizerに渡す。

### 2.2 FSQ（有限スカラー量子化）

FSQはVQ-VAEを簡素化した量子化手法で、コードブック検索を排除する。

**動作プロセス**:
1. コンテンツ特徴量 `fc` をD次元空間に射影（下方射影: 768 → 3）
2. 各次元を最近傍整数に丸める: 範囲 `[-Ki, Ki]`、`Vi = 2Ki + 1`
3. 量子化されたD次元を元の次元に射影（上方射影: 3 → 768）

| パラメータ | 値 |
|---|---|
| FSQレベル | [5, 3, 3] |
| D（次元数） | 3 |
| K値 | K1=2, K2=1, K3=1 |
| コードブックサイズ V | 5 × 3 × 3 = **45** |
| 勾配伝搬 | ストレートスルー推定器（STE） |

### 2.3 コンテンツ-スタイル分離の3つの鍵

#### (a) テキスト監視（ASR損失）
- 系列変換ASR損失によるエンドツーエンド学習
- テキスト予測に必要な言語情報のみがFSQボトルネックを通過
- スタイル情報はASR予測に不要 → ボトルネック容量の制限により自動的に抑制

#### (b) コンパクトコードブック（45符号）
- CosyVoice 2のコードブック6561と比較して劇的に小さい
- AutoVCの直感: ボトルネック幅を狭くしてスタイル情報の通過を物理的に阻止

#### (c) 連続的な量子化前特徴量
- FSQ離散インデックスではなく、FSQ **直前** の連続表現を使用
- SoftVCの「ソフトユニット」の着想に基づく
- 連続特徴量はVevoの離散トークンよりもスタイル情報が少ない（表5）
- FSQインデックスを直接使用 → WER 123.5%と壊滅的（表6）

### 2.4 CosyVoice 2 / Vevo との比較

| 観点 | StyleStream | CosyVoice 2 | Vevo |
|---|---|---|---|
| 学習監視 | ASR損失（系列変換） | ASR損失 | 自己教師あり |
| 量子化 | FSQ、コードブック45 | FSQ、コードブック6561 | VQ-VAE、コードブック32 |
| コンテンツ表現 | 連続的な量子化前特徴量 | 離散トークン | 離散トークン |
| 話者分類精度 | 3.50% | 25.00% | 23.00% |
| ストリーミング | 対応（約1秒） | 対応 | 非対応 |

## 3. Stylizer（拡散トランスフォーマー）

### 3.1 DiTアーキテクチャ

| パラメータ | 値 |
|---|---|
| トランスフォーマー層数 | 16 |
| 隠れ層サイズ | 768 |
| FFNサイズ | 3072 |
| 条件統合方式 | adaLN-Zero |

**入力構成**: 以下をチャネル方向に結合
1. ノイズ付きメルスペクトログラム `xt` ∈ R^{100 × T}
2. コンテキストメル `(1-m) ⊙ x1` ∈ R^{100 × T}
3. コンテンツ特徴量 `fc` ∈ R^{Dc × T}

**adaLN-Zeroによる条件付け**:
1. フロー時間ステップ `t ~ U[0,1]` を正弦波位置エンコーディングでベクトル化
2. スタイル埋め込み `e` と加算: `c = emb(t) + e`
3. 各DiT層で `c` から `(γ1, β1, α1, γ2, β2, α2)` をMLPで生成
4. 自己注意機構/FFN出力にスケール/シフト/ゲートを適用
5. ゲート（`α`）はゼロ初期化 → 学習初期は恒等写像として動作

### 3.2 条件付きフローマッチング（CFM）

**最適輸送フローパス**:
```
ψ_t(x) = (1-t) * x0 + t * x1,    x0 ~ N(0,I),  t ~ U[0,1]
```

**ノイズ付きメルスペクトログラム**:
```
xt = (1-t) * x0 + t * x1
```

**CFM損失**（マスク領域のみで計算）:
```
L_CFM(θ) = E_{t,x1,x0,m} [ || m ⊙ (v̂ - (x1 - x0)) ||² ]
```

目標速度場 `x1 - x0` は時刻 `t` に依存しない定数（最適輸送パスの特性）。

**オイラーサンプリング**（推論時）:
```
x0 ~ N(0, I)
dt = 1/NFE = 1/16
for i in range(16):
    t = i * dt
    v = v_θ,CFG(xt, t; 条件)
    x_{t+dt} = xt + v * dt
```

### 3.3 スペクトログラムインペインティング

| | 学習時 | 推論時 |
|---|---|---|
| マスク率 | 70-100%ランダム | ソース部分を100%マスク |
| コンテキスト | 同一発話の非マスク部分 | ターゲット発話のメル |
| セグメント長 | 6秒 | ターゲット＋ソースの結合 |

### 3.4 分類器不使用ガイダンス（CFG）

```
v_CFG = (1 + α) * v_θ(xt, t; c) - α * v_θ(xt, t; ∅)
```

| パラメータ | 値 |
|---|---|
| CFG強度 α | 2 |
| コンテンツ特徴量ドロップ率 | 0.2 |
| コンテキストメルドロップ率 | 0.3 |
| スタイル埋め込みドロップ率 | 0.3 |

## 4. スタイルエンコーダ

### 4.1 アーキテクチャ

```
ターゲット音声 → 凍結WavLM各層 → 学習可能な重み付き合算 → TDNN → 注意付き統計プーリング → スタイル埋め込み e
```

| コンポーネント | 仕様 |
|---|---|
| ベースモデル | `microsoft/wavlm-base-plus-sv`（凍結） |
| WavLM隠れ層次元 | 768 |
| WavLM層数 | 13（トランスフォーマー12層 + CNN 1層） |
| 層集約 | 学習可能な係数による加重平均 |
| 後段ネットワーク | TDNN（ECAPA-TDNN参照） |
| プーリング | 注意付き統計プーリング |
| 出力次元 | 768（推定、DiT隠れ層サイズと一致） |
| 学習方法 | DiTとエンドツーエンド（CFM損失） |

### 4.2 DiTへの統合

- スタイル埋め込み `e` はグローバルベクトル（フレームごとではない）
- フロー時間ステップ埋め込みと加算 → adaLN-Zeroで各DiT層に注入
- 入力チャネルに連結されるのではなく、**条件ベクトルとして各層に注入**

### 4.3 アブレーション: スタイルエンコーダの重要性（表6）

| | S-SIM | A-SIM | E-SIM | WER |
|---|---|---|---|---|
| スタイルエンコーダあり | 0.837 | 0.626 | 0.733 | 10.7% |
| スタイルエンコーダなし | 0.775 | 0.509 | 0.653 | 15.3% |

スタイルエンコーダの除去により全スタイル指標が大幅劣化。コンテキストメルだけではグローバルスタイルの捕捉が不十分。

### 4.4 推論時の最適化

- スタイル埋め込みはターゲット音声から**事前に1回だけ抽出**し、推論中キャッシュして再利用
- ストリーミング推論のチャンクごとの処理時間 `tproc` には含まれない

## 5. ボコーダ（因果的Vocos）

### 5.1 アーキテクチャ

- **ベース**: Vocos（ISTFTベースの周波数領域ボコーダ）
- **変更点**: ConvNextブロック内の**全畳み込み層を因果的畳み込みに変換**
  - 深さ方向畳み込み: 左側のみパディング（将来フレーム非参照）

## 7. ストリーミング設計

### 7.1 チャンク因果的注意機構

各チャンクは**自チャンク＋全過去チャンク**にのみ注意を向ける。未来チャンクは参照しない。

| コンポーネント | ストリーミング対応 |
|---|---|
| Destylizer HuBERT | 凍結解除 + チャンク因果的注意 |
| Destylizer Conformer | チャンク因果的注意 + 因果的畳み込み |
| Stylizer DiT | チャンク因果的注意 |
| ボコーダ | 因果的畳み込み（最初から因果的） |

**チャンクサイズ**: 600ms = 30フレーム（50Hz）

### 7.2 推論パイプライン

- **固定長ターゲット発話（5秒）**: コンテキストとして毎チャンク供給
- **リングバッファ（5秒）**: ソースのコンテンツ特徴量をチャンク単位で蓄積
- **スタイル埋め込み**: 事前抽出・キャッシュ（毎チャンク再計算不要）

### 7.3 レイテンシ

```
L = チャンクサイズ + 処理時間 = 600ms + 412.7ms ≈ 1013ms
```

ストリーミング条件: 処理時間（412.7ms）< チャンクサイズ（600ms）→ 成立

### 7.4 GPU別処理時間

| GPU | 処理時間（600msチャンク） | ストリーミング可否 |
|---|---|---|
| RTX A6000 | 約0.43秒 | 可能（余裕 約170ms） |
| RTX 4060 ノートPC | 約0.67秒 | **不可**（600msチャンクでは超過） |

RTX 4060では800msチャンク（処理時間0.67秒 < 0.8秒）でストリーミング可能だが、レイテンシは約1.5秒。

### 7.5 オフラインからストリーミングへの変換

| 段階 | 内容 |
|---|---|
| 段階1 | ストリーミングDestylizer: オフラインチェックポイントから初期化 → MSE蒸留 |
| 段階2 | ストリーミングStylizer: オフラインチェックポイントからウォームスタート → ストリーミングDestylizer出力で学習 |

### 7.6 チャンクサイズと品質のトレードオフ（表2）

| チャンク（ms） | WER(%) | S-SIM | A-SIM | E-SIM | UTMOS |
|---|---|---|---|---|---|
| 200 | 19.8 | 0.797 | 0.586 | 0.704 | 2.33 |
| 400 | 15.0 | 0.806 | 0.597 | 0.720 | 2.69 |
| **600** | **13.6** | **0.811** | **0.602** | **0.734** | **2.85** |
| 800 | 12.7 | 0.816 | 0.607 | 0.745 | 2.94 |
| 1000 | 12.6 | 0.820 | 0.613 | 0.751 | 3.00 |

600msが品質とレイテンシの最適バランス。200ms→600msで急激に改善し、600ms以降は収穫逓減。

**主な成果**:
- 全スタイル類似度指標で最高性能（S-SIM、A-SIM、E-SIM、S-SMOS、A-SMOS、E-SMOS）
- 特にE-SIM: +0.115、E-SMOS: +0.93の大幅改善（Vevo比）
- オフラインWER 9.2%で最高の明瞭性
- N-MOSのみCosyVoice 2.0に僅差で劣る（3.42 対 3.47）

### 8.4 評価データセット（StyleStream-Test）

- **ソース**: 300発話（ESD + GLOBE-test + LibriTTS-test-clean）
- **ターゲット**: 10発話（5感情: 喜び、怒り、悲しみ、恐怖、穏やか ＋ 5アクセント: 英国、米国、インド、アラビア、中国語）
- **合計**: 3,000ペア

## 10. 再現実装計画

### 10.1 必要な外部モデル

| モデル | HuggingFace ID | 用途 |
|---|---|---|
| HuBERT-Large-ASR | `facebook/hubert-large-ls960-ft` | Destylizer入力（第18層） |
| WavLM-Base-Plus-SV | `microsoft/wavlm-base-plus-sv` | スタイルエンコーダ |
| Vocos | `gemelo-ai/vocos` | ボコーダのウォームスタート |
| Whisper-large-v3 | `openai/whisper-large-v3` | 評価（WER） |

| emotion2vec | `ddlBoJack/emotion2vec` | 評価（E-SIM） |

### 10.2 論文で明記されていない実装詳細

| 項目 | 推測 |
|---|---|
| Conformer隠れ層次元 | 768（ASRデコーダと同一） |
| Conformer畳み込みカーネル | 31（Conformer原論文デフォルト） |
| 注意ヘッド数 | 12（768/64） |
| メルn_fft | 1024（Vocosデフォルトに準拠） |
| メル周波数範囲 | 0〜8000 Hz |
| DiT入力射影 | (100+100+768) → 768 の線形層 |
| スタイル埋め込み次元 | 768（DiT隠れ層サイズに一致） |
| CFGヌル条件 | ゼロベクトルで置換 |
| ASR損失種類 | CTCまたは注意機構ベース（CosyVoice 2参考） |
| マスク戦略 | 連続区間マスク（インペインティング目的から推測） |

### 10.3 推奨実装順序

| フェーズ | 内容 | 推定期間 | 優先度 |
|---|---|---|---|
| 1 | データ前処理パイプライン | 1〜2週間 | 高 |
| 2 | Destylizer（オフライン） | 2〜3週間 | **最高** |
| 3 | Stylizer / DiT（オフライン） | 3〜4週間 | **最高** |
| 4 | ボコーダ（因果的Vocos） | 1〜2週間 | 高 |
| 5 | ストリーミング対応 | 2〜3週間 | 低（オフライン完成後） |
| 6 | 評価パイプライン | 1〜2週間 | 中（並行可） |

### 10.4 推奨Pythonライブラリ

**コア**: `torch>=2.1`, `torchaudio>=2.1`, `transformers>=4.36`, `accelerate`, `einops`
**学習**: `wandb`/`tensorboard`, `datasets`（HuggingFace）
**評価**: `resemblyzer`, `jiwer`, `scipy`
**参考実装**: F5-TTS（DiT+CFM）、Vocos（ボコーダ）、CosyVoice 2（FSQ+ASR）

### 10.5 データ入手

| データセット | 規模 | 入手性 |
|---|---|---|
| Emilia-EN | 5万時間 | HuggingFace（オープン、数TB） |
| LibriTTS | 585時間 | OpenSLR（完全オープン） |
| MSP-Podcast | 数百時間 | UTDallas（研究利用申請必要） |
| GLOBE | 数百時間 | 論文記載の方法 |

### 10.6 推奨する最小実験構成

**LibriTTS（585時間）のみでフェーズ1〜4を実施し、オフライン版のWERとスタイル類似度を検証**。
これなら2〜4台のGPUで1〜2週間以内に最初の結果が得られる。

### 10.7 主要リスクと対策

| リスク | 対策 |
|---|---|
| ASR損失設計が不明確 | CTC損失で開始 → 不足なら交差注意デコーダに切替 |
| Emilia 5万時間が巨大 | LMG（1.3千時間）で全パイプラインを先に検証 |
| メルスペクトログラム不一致 | Vocos公式設定をベースにホップ=320のみ変更 |
| DiT+CFM学習不安定 | F5-TTS実装を参考。NFE=4の高速実験で先にイテレーション |
| MSP-Podcastの入手困難 | ESD（感情音声データセット）で代替 |
