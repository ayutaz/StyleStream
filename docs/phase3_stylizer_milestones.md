## 概要

Stylizerは本システムの中核コンポーネントであり、Destylizerが抽出したコンテンツ特徴量とターゲットのスタイル情報からメルスペクトログラムを生成する。16層Diffusion Transformer（DiT）にConditional Flow Matching（CFM）を組み合わせ、スペクトログラムインペインティングの枠組みで動作する。

**全体推定期間**: 3〜4週間（オフライン版）
**推定計算コスト**: 1,000〜1,400 GPU時間（8台 A6000、400kステップ）

## M3.1: DiTブロック実装（adaLN-Zero付き）

### タスク名
Adaptive Layer Norm Zero付きDiffusion Transformerブロックの実装

### 具体的な成果物
- `src/stylizer/dit_block.py`: 単一DiTブロックの実装
- `src/stylizer/adaln_zero.py`: adaLN-Zeroモジュール
- `src/stylizer/dit.py`: 16層DiT全体のモデルクラス
- `tests/test_dit_block.py`: 単体テスト

### 設計詳細

#### DiTブロック構造
各DiTブロックは以下の順序で処理を行う:

1. **adaLN-Zero前段（自己注意機構）**
   - 条件ベクトル `c = emb(t) + e` からMLPで `(gamma_1, beta_1, alpha_1)` を生成
   - LayerNormの出力に `gamma_1`（スケール）と `beta_1`（シフト）を適用
   - Multi-Head Self-Attention（ヘッド数12、ヘッド次元64と推定）
   - 出力に `alpha_1`（ゲート）を乗算

2. **adaLN-Zero後段（FFN）**
   - 条件ベクトル `c` から `(gamma_2, beta_2, alpha_2)` を生成
   - LayerNormの出力に `gamma_2` / `beta_2` を適用
   - FFN（768 → 3072 → 768、活性化関数はGELU推定）
   - 出力に `alpha_2` を乗算

3. **ゼロ初期化**
   - `alpha_1`, `alpha_2` のゲート出力MLPの最終層をゼロ初期化
   - 学習初期にDiTブロックが恒等写像として機能 → 学習安定化

#### adaLN-Zero条件付けMLP
```
c (768次元) → SiLU → Linear(768, 768×6) → 6分割 → (gamma_1, beta_1, alpha_1, gamma_2, beta_2, alpha_2)
```
各出力は768次元ベクトル。

#### 時間ステップ埋め込み
- フロー時間 `t ~ U[0,1]` を正弦波位置エンコーディングでベクトル化
- DiT-originalと同様に周波数を対数スケールで配置
- 出力次元: 768

### 受け入れ基準
- [x] 入力テンソル `(B, T, 768)` と条件ベクトル `(B, 768)` を受け取り、同形状を出力
- [x] adaLN-Zeroのゲート `alpha` がゼロ初期化されていることを確認（初期出力がゼロ）
- [x] 16層積み重ね時の総パラメータ数が論文規模と整合（推定約100〜130M）
- [x] 勾配が全層に正常に伝搬することを確認（nanなし）
- [x] F5-TTSのDiTブロック実装と構造比較を実施し、差異を文書化

## M3.2: 入力処理パイプライン（チャネル結合・射影層）

### タスク名
DiTへの入力前処理（3系統のチャネル結合と次元射影）

### 具体的な成果物
- `src/stylizer/input_projection.py`: 入力処理モジュール
- `src/stylizer/output_projection.py`: DiT出力からメルへの射影
- `tests/test_input_projection.py`: 単体テスト

### 設計詳細

#### 入力チャネル結合
3つの入力をチャネル方向（特徴量次元）に結合する:

| 入力 | 次元 | 説明 |
|---|---|---|
| ノイズ付きメル `x_t` | 100 x T | CFMで生成されるノイズ付きスペクトログラム |
| コンテキストメル `(1-m) * x1` | 100 x T | マスクされていない部分のターゲットメル |
| コンテンツ特徴量 `fc` | 768 x T | Destylizerの出力（連続表現） |

結合後の次元: **(100 + 100 + 768) = 968 x T**

#### 入力射影層
結合テンソルをDiTの隠れ層次元に変換:
```
Linear(968, 768)  # (100+100+768) → 768
```

論文では射影層の詳細は未記載だが、DiTの標準的な設計として線形射影を採用する。F5-TTSではConv1dを使用する例もあり、比較検討する。

#### 出力射影層
DiT最終出力からメルスペクトログラムの速度場を出力:
```
adaLN → Linear(768, 100)  # 768 → メル100ビン
```

DiT-originalに従い、最終層にもadaLNを適用後に線形射影する。

### 受け入れ基準
- [x] 3系統の入力を正しくチャネル結合し、形状 `(B, T, 968)` を生成
- [x] 射影後の出力形状が `(B, T, 768)` であること
- [x] 出力射影が速度場 `(B, T, 100)` を正しく出力すること
- [x] 時間軸Tが可変長に対応すること（50Hz、6秒 = 300フレームを基準）
- [x] Destylizerのコンテンツ特徴量（768次元）が正しく結合されることを、実際のDestylizer出力で検証

## M3.3: スタイルエンコーダ（WavLM-TDNN）実装

### タスク名
凍結WavLM-Base-Plus-SVとTDNNによるスタイル埋め込み抽出器の実装

### 具体的な成果物
- `src/stylizer/style_encoder.py`: スタイルエンコーダ全体
- `src/stylizer/tdnn.py`: TDNNブロック（ECAPA-TDNN参照）
- `src/stylizer/attentive_stat_pool.py`: 注意付き統計プーリング
- `tests/test_style_encoder.py`: 単体テスト

### 設計詳細

#### パイプライン
```
ターゲット音声波形 (16kHz)
  → 凍結WavLM-Base-Plus-SV（全13層の隠れ状態を取得）
  → 学習可能な重み付き合算（13層 → 1つの特徴量系列）
  → TDNN（時間畳み込みネットワーク）
  → 注意付き統計プーリング（可変長系列 → 固定長ベクトル）
  → スタイル埋め込み e ∈ R^768
```

#### WavLM層集約
- `microsoft/wavlm-base-plus-sv` の13層（CNN 1層 + Transformer 12層）全てから隠れ状態を取得
- 学習可能なスカラー重み `w_i` （i=0,...,12）をsoftmaxで正規化し加重平均
- WavLMの重み自体は凍結、層集約の重み `w_i` のみ学習

#### TDNN（Time Delay Neural Network）
ECAPA-TDNNの設計を参考とする。論文では「TDNN」とのみ記載されているため、以下の構成を推定:

- 1次元畳み込みの積み重ね（複数の時間遅延コンテキスト）
- チャネル次元: 768 → 1024 → 768（推定）
- ECAPA-TDNNのSE-Res2Block構造の簡略版を採用

#### 注意付き統計プーリング（Attentive Statistics Pooling）
- フレームレベル特徴量に注意重みを計算
- 重み付き平均と重み付き標準偏差を結合
- 線形射影で最終的なスタイル埋め込み次元（768）に変換

- [x] WavLMの重みが凍結されていることを確認（`requires_grad=False`）
- [x] 層集約の重み `w_i` が学習可能であることを確認

- [x] アブレーション表6の性能差（スタイルエンコーダなし: S-SIM 0.775→0.837）を再現の目安とする
- [x] ECAPA-TDNNの参考実装（SpeechBrain等）と構造を照合

## M3.4: インペインティング機構（マスク生成・コンテキスト構築）

### タスク名
スペクトログラムインペインティングのためのマスク生成とコンテキスト構築

### 具体的な成果物

- `tests/test_inpainting.py`: マスク率の統計的検証テスト

### 設計詳細

#### 学習時のマスク生成
- 6秒（300フレーム@50Hz）のセグメントに対してマスクを生成
- **マスク率**: 70%〜100%の一様ランダムサンプリング
- **マスク形状**: 連続区間マスク（インペインティングの定義から推測）
  - セグメント内でランダムな開始位置から連続的にマスク
  - 複数の非連続マスクではなく、1つの連続区間を推奨（論文のインペインティング記述に基づく）
- **マスクテンソル**: `m ∈ {0, 1}^T`（1=マスク、0=非マスク）

#### 推論時のマスク構成
- ターゲット音声（5秒）: 非マスク（コンテキスト）
- ソース音声: 100%マスク（生成対象）
- 結合: `[ターゲットメル | ゼロ（ソース分）]`

#### コンテキストメルの構築
```
context_mel = (1 - m) * x1  # 非マスク部分のみ残す
```

学習時: 同一発話のメルスペクトログラムの非マスク部分
推論時: ターゲット音声のメルスペクトログラム全体

#### セグメントサンプリング戦略
- 発話から6秒の連続セグメントをランダムに切り出し
- 6秒未満の発話はパディングまたはスキップ
- バッチ内で同一長にするための処理

## M3.5: CFM学習ループ（OTパス・マスク領域損失）

### タスク名
Conditional Flow Matchingの最適輸送パスとマスク付き損失関数の実装

### 具体的な成果物
- `src/stylizer/cfm.py`: CFMの中核ロジック（OTパス、ノイズ付きメル生成、損失計算）
- `src/stylizer/trainer.py`: 学習ループ（1ステップの訓練処理）
- `tests/test_cfm.py`: CFM損失の数値的検証テスト

### 設計詳細

#### 最適輸送（OT）フローパス
線形補間による最もシンプルなOTパス:
```
x_t = (1 - t) * x_0 + t * x_1
```
- `x_0 ~ N(0, I)`: 標準正規ノイズ（メル次元100 x T）
- `x_1`: 真のメルスペクトログラム（ターゲット）
- `t ~ U[0, 1]`: 一様分布からサンプリング

目標速度場は時刻 `t` に依存しない定数:
```
u_t = x_1 - x_0
```

#### CFM損失（マスク領域のみ）
```
L_CFM = E_{t, x_1, x_0, m} [ || m * (v_hat - (x_1 - x_0)) ||^2 ]
```
- `v_hat = DiT(x_t, t, fc, context_mel, e)`: DiTが予測する速度場
- `m`: マスクテンソル（1=マスク領域、0=非マスク領域）
- 損失はマスク領域のピクセルのみで計算（非マスク領域は無視）

#### 学習1ステップの処理フロー
1. バッチからメル `x_1`、コンテンツ特徴量 `fc`、ターゲット音声を取得
2. マスク `m` を生成（M3.4のインペインティング機構）
3. コンテキストメル `(1 - m) * x_1` を計算
4. スタイル埋め込み `e` を抽出（M3.3のスタイルエンコーダ）
5. `t ~ U[0,1]` をサンプリング
6. `x_0 ~ N(0, I)` をサンプリング
7. `x_t = (1 - t) * x_0 + t * x_1` を計算
8. DiTで速度場 `v_hat` を予測
9. マスク付き損失を計算してバックプロパゲーション

### 受け入れ基準
- [x] OTパスの補間が数値的に正しいこと（t=0で `x_0`、t=1で `x_1`）
- [x] 損失がマスク領域のみで計算されていることを検証（非マスク領域の勾配がゼロ）
- [x] 損失値がnanやinfにならないことを確認（最低1000ステップ）
- [x] 学習初期の損失値が合理的な範囲にあること（adaLN-Zeroのゼロ初期化により、初期出力はゼロ→損失は `||x_1 - x_0||^2` のオーダー）
- [x] ミニバッチ学習が正常に動作すること（バッチサイズ64、6秒セグメント）
- [x] F5-TTSのCFM損失実装と数値的に一致すること

### 推定作業量
**3日**

### 依存関係
- **M3.1**: DiTブロック（速度場予測モデル）
- **M3.2**: 入力処理パイプライン

### 技術的リスクと対策

| リスク | 深刻度 | 対策 |
|---|---|---|
| CFM学習の不安定性（損失が発散） | 高 | F5-TTSの学習設定を忠実に再現。時間ステップのサンプリングにlogit-normal分布（SD3で提案）の採用も検討。まずはU[0,1]で開始 |
| マスク領域のみの損失でバッチ間の勾配スケールが異なる | 中 | マスク率に応じた正規化を実装。マスク領域のピクセル数で割る平均損失を使用 |
| メモリ不足（100ビン x 300フレーム x バッチ64） | 中 | gradient checkpointingを有効化。混合精度学習（bf16/fp16）を使用 |

## M3.6: CFG実装（条件ドロップ戦略）

### タスク名
Classifier-Free Guidanceの3条件独立ドロップ戦略の実装

### 具体的な成果物
- `src/stylizer/cfg.py`: CFGのドロップ戦略と推論時のガイダンス計算
- `tests/test_cfg.py`: ドロップ率の統計テスト、CFG推論テスト

### 設計詳細

#### 学習時の条件ドロップ
学習時の各ステップで、3つの条件を独立にランダムドロップする:

| 条件 | ドロップ率 | ヌル条件（置換先） |
|---|---|---|
| コンテンツ特徴量 `fc` | 0.2 | ゼロテンソル `(B, T, 768)` |
| コンテキストメル | 0.3 | ゼロテンソル `(B, T, 100)` |
| スタイル埋め込み `e` | 0.3 | ゼロベクトル `(B, 768)` |

各条件は**独立に**ドロップされるため、以下の組み合わせが学習時に出現する:
- 全条件あり: `0.8 * 0.7 * 0.7 = 39.2%`
- 全条件なし: `0.2 * 0.3 * 0.3 = 1.8%`
- その他の部分的ドロップ: 残りの59.0%

#### 推論時のCFGガイダンス
```
v_CFG = (1 + alpha) * v_theta(x_t, t; fc, ctx, e) - alpha * v_theta(x_t, t; null, null, null)
```
- `alpha = 2`（CFG強度）
- 各NFEステップで2回のフォワードパスが必要（条件ありと条件なし）
- 合計フォワードパス数: `NFE * 2 = 32`

#### バッチ化された推論の最適化
条件ありと条件なしのバッチを結合して1回のフォワードパスで処理する:
```
v_cond, v_uncond = DiT([x_t; x_t], [t; t], [fc; null], [ctx; null], [e; null]).chunk(2)
v_cfg = (1 + alpha) * v_cond - alpha * v_uncond
```

### 受け入れ基準
- [x] 学習時のドロップ率が統計的に正しいこと（10万サンプルで各ドロップ率の誤差5%以内）
- [x] CFG alpha=0 のとき、通常の条件付き推論と一致すること
- [x] CFG alpha=2 のとき、出力が条件付き方向に増幅されること（定性的検証）
- [x] バッチ化CFG推論が非バッチ版と数値的に一致すること
- [x] ヌル条件（ゼロテンソル）がDiTのフォワードパスで正常に処理されること

### 推定作業量
**2日**

### 依存関係
- **M3.5**: CFM学習ループが動作すること

### 技術的リスクと対策

| リスク | 深刻度 | 対策 |
|---|---|---|
| ヌル条件の定義が推測 | 中 | ゼロベクトル置換を基本とする。F5-TTSでの実装を確認し、学習可能なヌルトークンの採用も検討 |
| CFG強度alpha=2が強すぎる可能性 | 中 | alpha=0, 1, 2, 3 での比較実験を計画。UTMOSで音質を定量評価 |
| 推論時の計算コスト倍増 | 低 | バッチ化による最適化を実装。GPU メモリに余裕があればバッチサイズ2倍で対応 |
| コンテキストとスタイルのドロップ率が同一（0.3）→ 同時ドロップの扱い | 低 | 論文記載通り独立にドロップ。コンテキストとスタイルが同時にドロップされるケース（確率9%）も学習に含まれる |

## M3.7: 推論パイプライン（オイラーサンプリング）

### タスク名
オイラー法によるCFMサンプリングとエンドツーエンド推論パイプラインの実装

### 具体的な成果物
- `src/stylizer/sampler.py`: オイラーサンプリング実装
- `src/stylizer/inference.py`: エンドツーエンド推論パイプライン
- `tests/test_sampler.py`: サンプリングの数値テスト
- `scripts/run_inference.py`: 推論実行スクリプト

### 設計詳細

#### オイラーサンプリング
```
x_0 ~ N(0, I)           # 初期ノイズ
dt = 1 / NFE = 1 / 16   # ステップ幅

for i in range(NFE):     # NFE = 16
    t = i * dt
    v = v_CFG(x_t, t, fc, ctx, e)   # CFG付き速度場予測
    x_{t+dt} = x_t + v * dt          # オイラー更新
```

#### 推論時のデータフロー
1. ソース音声 → Destylizer → コンテンツ特徴量 `fc_source`
2. ターゲット音声 → メルスペクトログラム計算 → `mel_target`
3. ターゲット音声 → スタイルエンコーダ → スタイル埋め込み `e`
4. コンテキスト構築: `[mel_target | zeros(ソース長)]`
5. マスク構築: `[0...0 | 1...1]`（ターゲット=非マスク、ソース=マスク）
6. コンテンツ特徴量構築: `[fc_target | fc_source]`（時間軸結合）
7. オイラーサンプリング（16ステップ、CFG alpha=2）
8. マスク領域のメルを抽出 → 変換メルスペクトログラム

#### NFE（Number of Function Evaluations）の設定
- 論文デフォルト: NFE=16
- 高速テスト用: NFE=4（品質低下あり、開発中の高速イテレーション向け）
- 高品質: NFE=32（品質改善が飽和するか検証）

### 受け入れ基準
- [x] NFE=16でのオイラーサンプリングが数値的に安定（nan/infなし）
- [x] 生成メルスペクトログラムの値域が合理的であること（実データの統計と比較）
- [x] 非マスク領域（ターゲットコンテキスト部分）がサンプリングにより変更されないこと
- [x] NFE=4での高速テストが動作すること
- [x] 6秒入力に対するA6000での推論時間を計測（目標: 1回のフォワードパスで1秒未満）
- [x] 学習済みモデルで生成した音声をボコーダに通し、聴取可能な波形が得られること
- [x] 論文の目標値（WER < 10%、S-SIM > 0.85、オフライン）に向けた品質のベースラインを確立

### 推定作業量
**2日**

### 依存関係
- **M3.5**: CFM学習ループ（学習済みモデルの重みが必要）
- **M3.6**: CFG実装
- **フェーズ4（ボコーダ）**: 最終的な波形生成にはボコーダが必要（メルの定性検証は独立で可能）

### 技術的リスクと対策

| リスク | 深刻度 | 対策 |
|---|---|---|
| オイラー法のステップ数不足で品質劣化 | 中 | NFE=16を基本とし、NFE=32と比較。改善が見られればミッドポイント法やRK4の採用を検討 |
| 推論時のメモリ不足（長い入力） | 中 | ターゲット5秒+ソース可変長。最大入力長を制限（例: 30秒）。チャンク処理はストリーミングフェーズで対応 |
