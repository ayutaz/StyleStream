### 主要リスク
- **ALiBi + Conformer**: 実装例が少ない → 原論文を参照、注意マスクを外部化

- FSQ: STE（z + (round(z) - z).detach()）パターン、コードブック利用率・perplexity追跡
- ASRヘッド: CTC（デフォルト）とseq2seqデコーダ両対応、loss_typeで切替
- コンテンツ特徴量fc: Conformer出力（FSQ前）768次元、推論時はFSQ/ASR不使用

### 主要リスク

- StyleEncoder: WavLM(凍結)+学習可能層重み+TDNN+注意付き統計プーリング→768次元

- CFG: 3条件独立Bernoulliドロップ（content 0.2, context 0.3, style 0.3）, α=2
- 入力: [x_t(100) + context_mel(100) + content(768)] = 968 → Linear → 768
- 出力: FinalAdaLN → Linear → 100メルビン速度場

| ID | マイルストーン | 日数 | 依存 | 状態 |
|---|---|---|---|---|

| MS4.2 | ウォームスタート・LibriTTSファインチューニング（100kステップ） | 3〜5 | MS4.1 | 🔲 (requires training run) |
| MS4.3 | 音質評価（PESQ, STOI, UTMOS, Stylizer結合テスト） | 2〜3 | MS4.2 | 🔲 (requires trained model) |

### 主要リスク
- **メル仕様不一致**: Vocos公式（80ビン, ホップ256）→ StyleStream（100ビン, ホップ320）変更が必要
- **GAN学習不安定**: Vocos公式の損失重み比率をそのまま使用

### 主要設計判断
- CausalConv1d: 左パディング (kernel_size-1)*dilation、右パディング0で因果性保証
- ConvNeXt V2ブロック: 深さ方向因果畳み込み → LayerNorm → 点方向畳み込み → GELU → 残差接続
- ISTFTHead: Conv1d(hidden, 2*(n_fft//2+1), 1) → 振幅exp() + 位相 → 複素STFT → torch.istft
- MultiScaleDiscriminator: 3スケール(1,2,4) × 重み正規化Conv1d、特徴マッチング損失用中間特徴量出力
- VocoderLoss: LS-GAN + メル再構成L1(重み45.0) + 特徴マッチングL1(重み2.0)
- VocoderTrainer: BaseTrainer拡張、G/D交互更新、2オプティマイザ・2スケジューラ管理
- ウォームスタート: パラメータ形状マッチングで共通層のみ転送、不一致層はランダム初期化

### 主要設計判断
- チャンク因果マスク: ブロック下三角（チャンク内は全結合、過去チャンクのみ参照）
- KVキャッシュ: 層別FIFO 250フレーム上限、推論時は暗黙的因果性
- StreamingHuBERT: CNN因果化（左パディング）+ チャンク因果注意マスク、深層のみ凍結解除
- StreamingDestylizer: HuBERT射影(1024→768) + 因果Conformer、MSE蒸留で学習
- StreamingStylizer: RoPEオフセットをKVキャッシュ長で管理、CFG二重バッチ時のキャッシュスナップショット
- リングバッファ: ソースコンテンツ特徴量FIFO + ターゲットキャッシュ → Stylizer入力構築
- 推論パイプライン: ターゲット初期化(1回) → チャンク処理(600ms) → RTF計測

L = チャンクサイズ(600ms) + 処理時間(412ms) ≈ 1013ms
